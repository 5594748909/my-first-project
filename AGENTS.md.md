# 一、总体开发原则

- **合规优先**：AI输出内容无恐怖、暴力、低俗、误导性信息，严格适配青少年模式过滤规则

- **玩法适配**：AI逻辑贴合海龟汤核心规则，仅输出**是、否、无关紧要**三类标准回复（特殊提示场景除外）

- **低延迟**：AI接口响应≤800ms，不影响游戏流畅度，支持断线兜底

- **可配置**：AI模型参数、回复阈值、审核规则支持后台灵活调整，无需重启服务

- **低成本**：选用轻量大模型/私有化部署方案，控制调用成本，支持缓存复用

# 二、核心AI模块开发细则

## 2.1 AI自动判题模块（单人模式核心）

### 2.1.1 功能描述

针对单人闯关模式，用户输入文字提问后，AI结合题目汤面、汤底、关键词库，快速判定并返回标准回复，替代人工判题，实现无人值守闯关。

### 2.1.2 核心开发要求

- **输入参数**：题目ID、用户提问文本（≤20字）、用户ID、模式标识（单人/AI汤主）

- **输出规则**：仅返回**是、否、无关紧要**三种结果，禁止额外话术、不泄露汤底线索

- **判定逻辑**：
            

    - 第一层：关键词精准匹配（匹配题库预设肯定/否定关键词，直接返回结果）

    - 第二层：语义相似度判定（AI理解提问意图，对比汤底核心逻辑，模糊匹配判定）

    - 第三层：兜底规则（无法判定时统一返回“无关紧要”，不影响推理节奏）

- **优化要求**：支持错题反馈收集，用户标记判题错误后，同步至后台人工修正关键词库

- **性能要求**：单次判题耗时≤500ms，支持1000次/分钟并发调用

### 2.1.3 接口规范

请求方式：POST；统一返回格式：`{code:200, msg:'success', data:{result:'是/否/无关紧要', error:false}}`

## 2.2 AI智能汤主模块（多人模式扩展）

### 2.2.1 功能描述

多人联机模式下，支持用户选择**AI汤主**替代真人房主，自动回复所有玩家提问，支持无人联机、单人练手场景，解决真人汤主短缺问题。

### 2.2.2 核心开发要求

- **房间适配**：AI汤主房间支持2-8人加入，逻辑同真人汤主，全员可见回复记录

- **回复规则**：严格遵循海龟汤规则，不额外提示、不偏题、不提前泄露汤底，拒绝引导性话术

- **特殊能力**：
            

    - 支持超时兜底：玩家1分钟无提问，AI发送轻量化引导语（如：可以从人物身份、事件原因开始提问哦）

    - 支持胜负判定：玩家完整说出汤底后，AI自动识别并判定获胜，结束游戏

    - 防骚扰过滤：忽略无意义提问、重复提问、违规言论，直接返回“无关紧要”

- **模式切换**：支持AI汤主与真人汤主一键切换，房间数据无缝同步

## 2.3 AI内容审核模块（UGC+聊天合规）

### 2.3.1 功能描述

对接AI内容审核能力，实现UGC投稿、房间聊天、用户评论的全流程智能审核，过滤违规内容，降低人工审核压力，保障平台合规。

### 2.3.2 核心开发要求

- **审核场景**：
            

    - UGC投稿审核：汤面、汤底、标签全文审核，判定是否违规、是否适配青少年模式

    - 实时聊天审核：多人房间公屏聊天、提问内容实时过滤，拦截违规言论

    - 用户信息审核：昵称、头像、个性签名合规校验

- **审核维度**：恐怖血腥、低俗色情、暴力违法、敏感政治、广告引流、侵权内容

- **分级处理**：
            

    - 高危违规：直接拦截、驳回投稿、禁言账号，同步人工复核

    - 中危违规：标记待复核，限制内容展示权限

    - 低危违规：提示修改，不拦截正常操作

- **青少年适配**：自动识别少儿不宜内容，标记后屏蔽青少年用户访问

## 2.4 AI智能推荐模块（用户留存优化）

### 2.4.1 功能描述

基于用户行为数据，AI智能推荐适配题库，提升用户答题兴趣与留存，实现个性化内容分发。

### 2.4.2 核心开发要求

- **推荐依据**：用户答题历史、偏好分类、难度习惯、答题时长、正确率

- **推荐规则**：
            

    - 新手用户：优先推荐入门级、经典类、温情类题库

    - 进阶用户：推荐同难度、同分类题库，穿插跨品类题目

    - 高玩用户：推荐高难度、原创优质题库

- **更新机制**：用户每完成3题，实时更新推荐列表，避免重复推荐

- **兜底逻辑**：无行为数据用户，默认推荐热门、精选题库

# 三、技术选型与部署要求

- **模型选型**：选用轻量大模型（如通义千问轻量版、文心一言轻量版），禁止使用高延迟、高成本大模型

- **部署方式**：优先API调用，核心判题逻辑本地缓存优化，支持私有化部署备选

- **缓存策略**：高频题目、常见提问的AI判定结果缓存至Redis，有效期24小时，减少重复调用

- **异常兜底**：AI服务宕机/超时，自动切换为关键词匹配兜底方案，保证游戏不中断

- **日志埋点**：记录AI调用次数、响应耗时、判题准确率、审核结果，用于模型优化

# 四、测试验收标准

- AI判题准确率≥95%，无违规回复、无泄露汤底情况

- AI接口平均响应≤500ms，峰值延迟≤800ms，无卡顿、无超时

- 内容审核拦截率≥98%，无漏判、无误判合规内容

- AI汤主模式可正常运行整局游戏，胜负判定准确

- 青少年模式下，AI无违规内容输出，屏蔽所有敏感信息
